# 个人网站维护手册

> 网站地址：https://HanYiton.github.io
> 仓库地址：https://github.com/HanYiton/HanYiton.github.io
> 模板：luost26/academic-homepage（Jekyll）

## 核心逻辑

**改文件 → 提交 → 推送 → GitHub 自动构建 → 1~2 分钟后网站更新**

内容都是纯文本（`.md` / `.yml`），不用写代码。

## 两种改法

**A. GitHub 网页改（最简单）**：打开仓库 → 点开文件 → 点铅笔 ✏️ → 改 → 底部 Commit changes。

**B. 本地改**：在 `E:\个人\academic-homepage` 用 VS Code 改，然后在该目录终端执行：
```
git add -A
git commit -m "更新说明"
git push
```

## 常见改动

### 1. 改个人信息（姓名/简介/邮箱/教育/获奖）
文件：`_data/profile.yml`
- `short_bio:` 里是自我介绍（支持 HTML，`<strong>加粗</strong>`、`<p>段落</p>`）
- `education:` / `experience:` 每条格式（`logo` 可省略，省略不会有破图）：
```yaml
- name: 学校或单位名
  position: 学位/职位
  date: Sep. 2023 - Jun. 2027
```
- `awards:` 每条：
```yaml
- name: 奖项名称
  date: 2026
```

### 2. 加一条新闻动态
在 `_news/` 里**新建一个 `.md` 文件**（文件名随意，建议 `年份-简述.md`），内容：
```
---
title: >-
    这里写动态内容。可加链接：<a href="网址" target="_blank">链接文字</a>
date: 2026-07-20 10:00:00 +0800
---
```
> 新闻按 `date` 从新到旧自动排序，主页最多显示 10 条（可在 `_data/display.yml` 改 `num_news`）。

### 3. 加一篇论文
在 `_publications/年份/` 里新建 `.md`（没有对应年份文件夹就新建一个）：
```
---
title:          "论文标题"
date:           2026-01-01 00:01:00 +0800
selected:       true          # true 会显示在主页“Selected Publications”
pub:            "会议或期刊名"
pub_last:       ' <span class="badge badge-pill badge-publication badge-success">CCF-A</span>'  # 可选徽章
pub_date:       "2026"
authors:
  - N. Lin
  - Y. Han                    # 你的名字，会自动加粗（见 _data/authors.yml）
links:                        # 可选链接
  Paper: https://论文地址
  Code: https://代码地址
---
```
> 若出现新的合作者名字、想让它加粗或加超链接，去 `_data/authors.yml` 补一条。

### 4. 加社交/学术链接（GitHub、Google Scholar、CV 等）
文件：`_data/profile.yml`，把对应行前面的 `#` 去掉并填值：
```yaml
cv_link: /assets/我的CV.pdf     # CV：把 PDF 放进 assets/ 文件夹
gscholar: 你的GoogleScholarID   # 学者主页网址里 user= 后面那串
github: 你的GitHub用户名
```
（图标会自动出现在头像下方）

### 5. 换头像
用新图片覆盖 `assets/images/photos/portrait.jpg`（保持同名即可）。

### 6. 恢复「博客 / 项目展示」板块
编辑 `_data/navigation.yml`，把 Blog / Showcase 那几行前面的 `#` 去掉。
- 博客文章放 `_posts/`，文件名必须 `年-月-日-标题.md`
- 项目展示放 `_showcase/`

## ⚠️ YAML 注意事项（最容易踩的坑）
- **缩进用空格，不能用 Tab**；同一层级对齐。
- 值里有冒号 `:`、`#`、引号时，整个值用**双引号**包起来。
- 列表项用 `- ` 开头（减号后有空格）。
- 改完拿不准，可在本地用这条命令校验语法（Ruby 已装在 `C:\Ruby33-x64`）：
  ```
  C:\Ruby33-x64\bin\ruby.exe -ryaml -e "YAML.load_file('_data/profile.yml'); puts 'OK'"
  ```

## 看构建是否成功
推送后，仓库顶部 **Actions** 标签有 “pages build and deployment” 任务：
- 🟢 绿勾 = 成功，刷新网站即可（强刷 Ctrl+F5）
- 🔴 红叉 = 出错，点进去看日志，通常是 YAML 缩进/引号问题
